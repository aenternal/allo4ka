# Allo4ka 2.0

> Мультиплатформенный бот для VK и Telegram с админкой, авторизацией, фоновыми voice-задачами, Alembic-миграциями и прегенерацией голосового кэша.

## Что реализовано

- Интеграция с **VK** и **Telegram** через единый сервис обработки входящих сообщений.
- Фоновая генерация голосовых ответов через **Celery** + **Redis**.
- Прегенерация **голосовых** вариантов ответов, которую можно запускать из админки.
- Веб-админка с авторизацией по логину/паролю.
- Админ API для управления триггерами, ответами, связями `trigger ↔ response`, фрагментами и просмотром voice-кэша.
- **Alembic**-миграции для схемы БД и стартового наполнения.
- Docker Compose для запуска всего стека: PostgreSQL, Redis, admin API, worker и bots.

## Авторизация в админке

Используются переменные окружения:
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `ADMIN_SESSION_SECRET`

После логина доступна веб-админка по адресу `/admin`.

## Прегенерация voice

Из админки можно поставить в очередь задачу прегенерации voice-кэша:
- для всех сохранённых ответов с `voice_enabled=true`;
- для N случайно собранных фраз из `response_fragments`.

Текстовые заготовки отдельно не сохраняются. Оптимизация работает через `audio_cache`: в нём лежит текст и путь к уже готовому voice-файлу.

## Мат-фильтр и кастомные триггеры

- Сначала бот проверяет уникальные DB-триггеры и связанные с ними ответы.
- Если кастомный триггер не найден, включается универсальный фильтр русского мата.
- Если сработал мат-фильтр, бот собирает реплику из `response_fragments`.

## Запуск через Docker Compose

```bash
cp .env.example .env
docker compose up --build
```

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .
cp .env.example .env
python run_admin.py --init-db
python run_admin.py
```

В отдельных терминалах:

```bash
celery -A app.tasks.celery_app.celery_app worker -l info
python run_bots.py
```
